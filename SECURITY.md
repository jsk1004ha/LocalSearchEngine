# Security Policy

## Supported Versions

이 프로젝트는 `VERSION`에 기록된 최신 버전을 기준으로 유지보수합니다.

## Reporting a Vulnerability

보안 취약점(원격 코드 실행, 데이터 유출, SSRF/취약한 웹 fetch 등)을 발견했다면 **공개 이슈에 상세 내용을 올리지 말아 주세요**.

권장 절차:

1. 재현 단계(최소 재현), 영향 범위, 예상되는 완화책을 정리합니다.
2. 저장소에 비공개 신고 채널(예: GitHub Security Advisories)이 설정되어 있다면 그 채널로 신고합니다.
3. 비공개 채널이 없다면, 공개 이슈에는 **“보안 이슈 신고 채널 문의”** 정도만 올리고 상세 내용은 공유하지 않습니다.

## Security Notes

- 웹 검색/웹 fetch는 외부 입력을 다루므로, 타임아웃/바이트 제한/격리 옵션을 우선 유지합니다.
- 웹 fetch 파서를 변경하는 PR은 악성 HTML/PDF 입력을 고려한 방어(크기 제한, 파싱 실패 처리, 로깅)를 포함해 주세요.
